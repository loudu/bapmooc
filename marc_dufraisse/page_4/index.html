BASIC PRINCIPLES

Computers work on the principle of number manipulation. People commonly use a decimal (base 10) numbering system, that is one counts from 0 to 9 before adding another digit. Computers use a simpler binary numbering system (base 2). Counting in binary is similar to counting in decimal, but there are only two values for a specific digit: 0 and 1. For example, counting from 0 to 3 is: 00, 01, 10, 11. 

Bits and bytes

Inside the computer, binary numbers are represented using bits and bytes. A bit is the implementation of a binary single digit in digital electronic circuitry used by almost all modern computers. A byte is a group of 8 bits (also called octet). 8-bit is a convenient power of two permitting the values 0 through 255 or -128 through +127 for signed numbers. Negative numbers uses two's complement: invert all the bits and then add 1 to the result, thus 00000001 still represents +1 and 11111111 represents -1.





Hexadecimal notation A more human-friendly representation of binary values is the hexadecimal numeral system (base 16). It uses 16 symbols : 0-9 and A-F to represent values 0 to 15. We often add the prefix 0x in order to avoid confusion with decimal numbers.

Characters and strings

Historically, the byte is also the number of bits used to encode a single character of text using the American Standard Code for Information Interchange (ASCII). It includes definitions for 128 characters (see ASCII printable characters). This convention enables computers to deal not only with numbers but also with texts. Texts are composed of strings of characters. The NULL character (code 00) indicates the end of a string. For example, the string "Hello!" is coded by the following sequence of 13 bytes: 48 65 6C 6C 6F 21 00.

 Programs and machine code 

A program generally consists of data, including numbers and text strings, and a sequence of instructions that tells the computer what to do. At the lower level, the instructions are binary numbers stored in the main memory. Each instruction is a binary machine code composed of one or more bytes that encodes the operation to be executed. The machine code depends on the processor. Each type of processor has it's own specific instruction set.
