The DATAPATH

The ALU and the bank of registers are tightly coupled using two source buses (S1, S2) and a destination bus (D). These buses are electrical wires with multiple connections which allow to transfer 8 bits in parallel. This connection scheme is called the datapath. The datapath is important since it structures the data flow as well as the organization of the silicon chip and it's efficiency.

Execution of a typical instruction

The execution of a typical instruction is broken into a small number of micro-operations, since it generally takes too long to do all steps in a single machine cycle.The Control Unit begins by reading a new instruction from the main memory (Fetch), and translates it in hardware commands such as the operation code for the ALU and the registers that will be used (Decode). The contents of the selected registers are read and held in buffers (Read). Then these binary values go though the ALU, which applies the selected operation (Execute).





The last step writes back the result in the selected destination register (Write). Finally, the Program Counter is incremented (+ 1) and a new cycle begins with the next instruction.

A reduced set of instructions

In order to meet the goals of simplicity and efficiency, SRISC8 is based on the Reduced Instruction Set Computer (RISC) architecture principles: Execute one instruction per cycle. Make all instructions the same size (8-bit in our case). Access memory only with load-store instuctions, the rest operate between registers. Support high-level languages by including the most occurrent operations

A complete set of instructions 

Even if the number of instructions is small, the set must include all the necessary operations for writing any possible program. This results in a total of 16 instructions classed in three groups: Arithmetic and logical instructions. Memory access instructions. Control flow instructions.

Arithmetic and logical instructions 

The first group includes mathematical and logical operations: ADD, INC, DEC, MOV, NOT, AND, LOR, XOR, ROT. These operations are performed by the ALU as shown by the animated illustration of the previous page. For example, the ADD instruction adds the content of register A with the content of another operand register and writes back the result in register A. The Zero, Carry and Negative flags are updated depending on the result of the operation.
