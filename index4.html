Experiement by yourself how works a computer

The SRISC8 simulator (next page) allows you to run our simple "Hello!" program example. In a second phase, you will be able to create your own assembly program. The simulator has three main parts: the processor on the top left, a 8-digit display and a keyboard on the bottom left, the main memory on the right 

Running the "Hello!" program



The memory has 32 bytes (0x20) and is initialized with the program at address 0x00 and the "Hello!" string at address 0x10. Each memory cell shows its address, its content both in binary and hexadecimal, and the assembly code for an instruction. 

 A click on the clear button clears all the memory cells. A click on the init button loads the "Hello!" example program
The processor part shows the content of all its registers, including the Program Counter, the Instruction Register and status flags. 
The reset button initializes the processor by resetting all its registers. The step button allows to execute the program instruction by instruction. The run button starts the execution while the stop button stops the execution. The current instruction is highlighted in blue in the main memory. 

 

Create your own assembly program

 You can input any assembly program from the instruction set, and the simulator will animate the program's execution 
 In order to create your own program, first clear memory with the clear button, then enter your program by editing memory cells. If you want to enter an instruction, first click on the edit button of the selected address. Then, click on an instruction key (for example ADD) to set the operation code (4 most significant bits), followed by an hexadecimal key (4 least significant bits). For a number or a character, just click on two hexadecimal keys
