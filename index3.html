Functions call and return

A program is generally much more complex than a small sequence of instructions. It is composed of an algorithm which is implemented using a set of functions, also called procedures, routines, subroutines, or methods depending on the programming language.
 A function is a sequence of instructions that perform a specific task, packaged as a unit. This unit can be used in programs wherever that particular task should be performed. Functions may be defined within programs or separately in libraries used by multiple programs 
Function call can be easily implemented by using the Jump and Link (JPL) instruction. It saves the return address in the Link Register (LR) and then goes to the address of the function's first instruction. 
The return to caller is simply implemented by branching to the address stored in LR. 

Local context and parameters

 However, in order to allow recursion (a function calling itself) and reentrancy (a function called by different programs), the function needs to save the context of the calling program (registers that will be used) and to pass parameters. This can be done by a sequence of instructions that stores these data in a stack: 

The stack is implemented as a contiguous area of memory pointed out by the Stack Pointer (SP). Each function call adds a new frame of data. When a function ends, the context is restored from the stack. 

What is a high-Level language

 Although assembly language allows to use all the processor's features, it is not used for large modern software projects. 
 High-level languages are prefered for most projects instead, allowing programmers to express their ideas more simply and directly. Programs are also portable on different types of computers. Some high-level languages, like C or C++, are compiled into machine code by a compiler. Some others, like Javascript, are directly executed by an interpreter.
